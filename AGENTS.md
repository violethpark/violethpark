# AGENTS.md — AI 에이전트 가이드라인

> 이 프로젝트에서 AI 에이전트가 작업할 때 반드시 따라야 할 규칙과 지침입니다.

---

## 프로젝트 개요

이 저장소는 **박형관(violethpark)**의 GitHub 프로필 저장소입니다.
RESUME*.md(원본 이력서 파일들)를 기반으로 CV.md와 README.md를 생성·관리합니다.

## 디렉토리 구조

```
violethpark/
├── AGENTS.md          # AI 에이전트 지침 (이 파일)
├── TASK.md            # 작업 순서 및 요구사항 정의
├── RESUME*.md         # 원본 이력서 (외부 사이트에서 변환, 한국어, 복수 파일 가능)
├── README.md          # GitHub 프로필 페이지 (영어, 기본)
├── README_ko.md       # GitHub 프로필 페이지 (한국어)
├── README_jp.md       # GitHub 프로필 페이지 (일본어)
├── CV.md              # CV (영어, 기본)
├── CV_ko.md           # CV (한국어)
├── CV_jp.md           # CV (일본어)
└── images/            # 이미지 자료
```

## 핵심 원칙

### 1. 사실 기반 원칙 (Factual Integrity)

- **RESUME*.md 파일들에 기재된 사실만 사용한다.**
- 경력, 학력, 프로젝트, 기술 스택 등 어떤 정보도 임의로 추가하거나 과장하지 않는다.
- 존재하지 않는 수상 이력, 자격증, 경력을 만들어내지 않는다.
- 불확실한 정보는 사용자에게 확인을 요청한다.

### 2. 원본 보존 원칙

- RESUME*.md의 **내용(사실관계)**은 변경하지 않는다.
- 수정 가능한 범위: 오탈자, 맞춤법, 마크다운 서식, 일관성
- 정보의 추가/삭제/변경이 필요하면 반드시 사용자에게 확인받는다.

### 3. 다국어 원칙

- **CV.md, README.md**의 기본 언어는 **영어**이다.
- 한국어 버전(`*_ko.md`)과 일본어 버전(`*_jp.md`)을 별도 파일로 제공한다.
- 각 문서 상단에 다른 언어 버전으로의 링크를 포함한다.
- **RESUME*.md**는 원본 언어(한국어)를 그대로 유지한다.
- 기술 용어, 고유명사 등은 모든 언어 버전에서 원어(영어) 표기를 허용한다.

### 4. CV 적합성 원칙

- RESUME*.md의 모든 정보가 CV/README에 들어갈 필요는 없다.
- CV 형식에 맞지 않는 정보는 **과감하게 제거**한다.
- 제거 대상 예시: 희망연봉, 전화번호, 상세 주소 등 개인 민감 정보
- RESUME*.md 원본은 변경하지 않으며, CV.md/README.md에서만 필터링한다.

### 5. 계정 노출 제한 원칙

- 생성되는 파일(CV.md, README.md 및 다국어 버전)에서 공식 계정(`violethpark`) 및 Outlook 이메일(`violethpark@outlook.com`)만 사용하여 노출하도록 한다.
- RESUME*.md 원본 파일은 변경하지 않는다 (원본 보존 원칙).

## 작업별 에이전트 지침

### Task 1: RESUME*.md 오탈자 검토

- `RESUME*.md` 패턴에 일치하는 모든 파일(RESUME.md, RESUME2.md 등)을 검토 대상으로 한다.
- 검토 항목:
  - 한글 맞춤법 및 띄어쓰기
  - 마크다운 문법 오류 (깨진 링크, 잘못된 헤딩 레벨 등)
  - 날짜/숫자 표기 일관성
  - 이미지 경로 유효성 (`images/` 폴더 내 파일과 대조)
  - 특수문자, 이모지 깨짐 여부
- 수정은 파일에 직접 반영한다.
- 의미가 달라질 수 있는 수정은 사용자에게 확인한다.

### Task 2: CV.md 생성

- RESUME*.md 파일들에서 핵심 정보를 추출하여 구조화한다.
- 여러 RESUME 파일이 있을 경우, 모든 파일의 정보를 통합하여 하나의 CV로 구성한다.
- **기본 언어: 영어**로 작성한다.
- CV 형식에 부적합한 정보(희망연봉, 전화번호, 상세 주소 등)는 제거한다.
- 권장 구성:
  1. 헤더 (이름, 한줄 소개)
  2. 연락처 / 링크
  3. 기술 스택 (카테고리별 정리)
  4. 경력 요약
  5. 주요 프로젝트
  6. 학력
  7. 자격증
  8. 자기소개 (간결하게 다듬기)
  9. 부록 (성격 검사 결과 등 — 이미지 포함 가능)
- 장황한 설명을 피하고 **간결하고 읽기 쉬운 포맷**을 사용한다.
- 추후 이미지 추가를 위한 플레이스홀더 또는 섹션을 마련한다.
- 문서 상단에 한국어(`CV_ko.md`), 일본어(`CV_jp.md`) 버전 링크를 포함한다.

### Task 3: README.md 생성

- CV.md를 기반으로 GitHub 프로필 페이지에 적합한 포맷으로 작성한다.
- **기본 언어: 영어**로 작성한다.
- CV 형식에 부적합한 정보(희망연봉, 전화번호, 상세 주소 등)는 제거한다.
- 활용 요소:
  - Emoji (👋, 🛠️, 💼, 🎓 등)
  - shields.io 배지 (기술 스택, 연락처 등)
  - GitHub Stats 카드 (선택)
  - 간결한 섹션 구분
- **분량:** 스크롤 1~2회 이내 (너무 길지 않게)
- **톤:** 전문적이면서 친근한 톤
- 문서 상단에 한국어(`README_ko.md`), 일본어(`README_jp.md`) 버전 링크를 포함한다.

### Task 4: CV.md 이미지/자료 보강

- 사용자에게 아래 항목을 **구체적으로** 요청한다:
  - 프로필 사진 (있다면)
  - 프로젝트 스크린샷 또는 데모 링크
  - 자격증 사본 이미지
  - 기타 포트폴리오 자료
- 기존 `images/` 폴더의 이미지를 CV.md에 적절히 배치한다.
- 이미지 경로는 `images/` 하위로 통일한다.

### Task 5: 다국어 버전 생성

- CV.md(영어)를 기반으로 한국어, 일본어 버전을 생성한다.
  - `CV_ko.md` — 한국어 번역
  - `CV_jp.md` — 일본어 번역
- README.md(영어)를 기반으로 한국어, 일본어 버전을 생성한다.
  - `README_ko.md` — 한국어 번역
  - `README_jp.md` — 일본어 번역
- 각 문서 상단에 다른 언어 버전으로의 링크를 상호 연결한다.
- 번역 시 기술 용어, 고유명사는 원어(영어)를 유지한다.
- 번역의 정확성을 위해 의역보다 직역을 우선하되, 자연스러운 표현을 사용한다.

## 코드/파일 작성 규칙

- 마크다운 파일은 UTF-8 인코딩을 사용한다.
- 이미지 파일명에 한글이 포함될 경우, 가능하면 영문으로 변경을 제안한다.
- 커밋 메시지는 한국어로 작성해도 무방하다.
- 파일 수정 시 변경 사유를 간단히 기록한다.
