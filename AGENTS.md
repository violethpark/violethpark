# AGENTS.md — AI 에이전트 가이드라인

> 이 프로젝트에서 AI 에이전트가 작업할 때 반드시 따라야 할 규칙과 지침입니다.

---

## 프로젝트 개요

이 저장소는 **박형관(violethpark)**의 GitHub 프로필 저장소입니다.
RESUME*.md(원본 이력서 파일들)를 기반으로 CV.md와 README.md를 생성·관리합니다.

## 디렉토리 구조

```
violethpark/
├── AGENTS.md          # AI 에이전트 지침 (이 파일)

├── RESUME*.md         # 원본 이력서 (외부 사이트에서 변환, 한국어, 복수 파일 가능)

```

## 핵심 원칙

### 1. 사실 기반 원칙 (Factual Integrity)

- **RESUME*.md 파일들에 기재된 사실만 사용한다.**
- 경력, 학력, 프로젝트, 기술 스택 등 어떤 정보도 임의로 추가하거나 과장하지 않는다.
- 존재하지 않는 수상 이력, 자격증, 경력을 만들어내지 않는다.
- 불확실한 정보는 사용자에게 확인을 요청한다.

### 2. 원본 보존 원칙

- RESUME*.md의 **내용(사실관계)**은 변경하지 않는다.
- 수정 가능한 범위: 오탈자, 맞춤법, 마크다운 서식, 일관성
- 정보의 추가/삭제/변경이 필요하면 반드시 사용자에게 확인받는다.

### 3. 다국어 원칙

- **CV.md, README.md**의 기본 언어는 **영어**이다.
- 한국어 버전(`*_ko.md`)과 일본어 버전(`*_jp.md`)을 별도 파일로 제공한다.
- 각 문서 상단에 다른 언어 버전으로의 링크를 포함한다.
- **RESUME*.md**는 원본 언어(한국어)를 그대로 유지한다.
- 기술 용어, 고유명사 등은 모든 언어 버전에서 원어(영어) 표기를 허용한다.

### 4. CV 적합성 원칙

- RESUME*.md의 모든 정보가 CV/README에 들어갈 필요는 없다.
- CV 형식에 맞지 않는 정보는 **과감하게 제거**한다.
- 제거 대상 예시: 희망연봉, 전화번호, 상세 주소 등 개인 민감 정보
- RESUME*.md 원본은 변경하지 않으며, CV.md/README.md에서만 필터링한다.

### 5. 계정 노출 제한 원칙

- 생성되는 파일(CV.md, README.md 및 다국어 버전)에서 공식 계정(`violethpark`) 및 Outlook 이메일(`violethpark@outlook.com`)만 사용하여 노출하도록 한다.
- RESUME*.md 원본 파일은 변경하지 않는다 (원본 보존 원칙).

## 코드/파일 작성 규칙

- 마크다운 파일은 UTF-8 인코딩을 사용한다.
- 이미지 파일명에 한글이 포함될 경우, 가능하면 영문으로 변경을 제안한다.
- 커밋 메시지는 한국어로 작성해도 무방하다.
- 파일 수정 시 변경 사유를 간단히 기록한다.
